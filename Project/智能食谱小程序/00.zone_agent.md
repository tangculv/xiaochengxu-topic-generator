# 智能食谱小程序 - Zone Agent 规则

## 项目性质

这是一个**产品规划与验证**阶段的项目，属于"小程序工厂"选题验证的一部分。

## 核心规则

### 1. 内容质量要求

所有涉及营养学、健康建议的内容：
- 必须标注"仅供参考"，不能替代专业医疗建议
- 必须有专家认证或参考文献来源
- 不能承诺治疗效果

### 2. 数据隐私

用户填写的健康信息（如慢性病、孕期等）：
- 仅用于生成个性化食谱
- 不用于其他目的
- 不与第三方共享
- 用户可随时删除

### 3. MVP验证导向

当前阶段专注于验证核心假设：
- 用户是否愿意分享健康信息？
- AI个性化推荐是否优于通用食谱？
- 用户留存率是否达到预期？

**避免范围蔓延**：V1.0及以后的功能（如食谱库检索、饮食打卡、社交分享）暂不讨论。

### 4. 开发准备

在开始开发前，需确认：
- [ ] MVP功能列表最终确认
- [ ] 技术选型明确（前端框架、后端栈、AI服务）
- [ ] 原型图通过评审
- [ ] 与先通（技术合作方）达成协作共识

### 5. 文档管理

- PRD 和 prototype.md 必须保持同步更新
- 任何需求变更需先更新 PRD，再调整原型
- 所有变更需有版本记录和变更说明

---

## 与全局规则的冲突处理

当本项目的 Zone Agent 规则与全局规则冲突时，**以本项目的 Zone Agent 规则为准**，原因：

- 本项目涉及健康内容，需要更严格的内容质量要求
- MVP 验证阶段需要控制范围，避免过早讨论扩展功能

---

## 审批流程

任何关键决策需经过以下流程：

1. 草案准备（PRD/原型/技术方案）
2. 内部评审（至少一人参与）
3. 外部反馈（如需，邀请先通或相关方）
4. 确认归档
