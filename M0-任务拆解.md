## 阶段 0：选定第一个小程序

> 第一个小程序的选择很关键——它决定了模板项目中组件库、布局模板、design tokens 的初始内容。好的第一个项目应该能覆盖生产线的主要能力（付费、分享、埋点、多种页面类型）。

### 0.1 从候选池中选择

| 任务 | 执行者 | 产出 |
|------|--------|------|
| 回顾现有候选选题（AI 智能食谱 96 分等） | Claude Code | 候选列表 |
| 对 top 候选做市场信号验证（搜一搜/小红书/知乎） | 你 | 市场信号评分 |
| AI 评估 + 立项闸门 8 项检查 | Claude Code | 立项报告 |
| **决策：选定第一个小程序** | **你** | 确认选题 |

**选择标准**（除了正常的选题标准外，第一个还需要）：
- 覆盖至少 2 种页面类型（如 form-to-result + list-detail）
- 有明确的付费场景
- 有自然的分享触发点
- 业务逻辑不过于复杂（把精力放在模板建设上，不是业务上）

### 0.2 生成 PRD

| 任务 | 执行者 | 产出 |
|------|--------|------|
| 基于选题生成标准化 PRD（第一个 PRD 是手写，后面的才用 PRD-RULES） | Claude Code | PRD.md |
| 审核 PRD，确认页面结构/付费/分享/埋点方案 | 你 | PRD 确认 |

> 第一个 PRD 不套规则生成——因为 PRD-RULES.md 还不存在。这个 PRD 写完后，反过来会成为 PRD-RULES 的参考样本。

## 阶段 2：设计系统 + 页面开发

> 这是 M0 最核心的阶段。做页面的过程就是沉淀设计系统的过程：先做出好看的页面，再把其中的视觉规则抽成 tokens，把重复的 UI 抽成组件。

### 2.1 建立 design tokens 初始版

| # | 任务 | 执行者 | 产出 | 验收 |
|---|------|--------|------|------|
| 2.1.1 | 确定主色和辅色（基于第一个小程序的调性） | Claude Code 建议，你确认 | 颜色方案 | 视觉舒适 |
| 2.1.2 | 建立 tokens 文件（颜色/字体/间距/圆角/阴影） | Codex | tokens.scss | 可引用 |

> tokens 的具体值在做页面过程中会调整，不用追求一次定稿。

### 2.2 逐页开发（按 PRD 中的页面列表）

每个页面的开发流程：

```
Claude Code 出页面设计说明（布局草图 + 组件说明）
  ↓
Codex 开发页面
  ↓
你在开发者工具预览
  ↓
如果某个 UI 模式是通用的 → 抽成组件
如果某个视觉参数需要统一 → 补进 tokens
```

| # | 任务 | 执行者 | 依赖 | 验收 |
|---|------|--------|------|------|
| 2.2.1 | 首页开发 | Codex | 1.6, 2.1 | 首页渲染正确、布局美观 |
| 2.2.2 | 核心功能页（如 form-to-result） | Codex | 2.2.1 | 输入→处理→结果流程走通 |
| 2.2.3 | 结果/详情页 | Codex | 2.2.2 | 结果展示正确、排版美观 |
| 2.2.4 | 个人中心页 | Codex | 1.4 | 用户信息展示正确 |
| 2.2.5 | 设置页 | Codex | 1.3 | 设置项可操作 |

> 每完成一个页面，回头看：有没有可以抽成通用组件的 UI？有没有需要补进 tokens 的值？

## 阶段依赖关系总览

```
阶段0（选题+PRD）
  ↓
阶段1（脚手架+基座）
  ↓
阶段2（设计系统+页面）──→ 阶段3.2（分享）──→ 阶段4（联调）
  ↓                      阶段3.3（埋点）──→    ↓
阶段3.1（支付，可并行）─────────────────────→ 阶段4
                                              ↓
                                          阶段5（规范沉淀）
                                              ↓
                                          阶段6（上线）
```

可并行的部分：
- 阶段 2 做页面的同时，阶段 3.1 可以独立做支付模块（不依赖页面）
- 阶段 3.2 和 3.3 可以在阶段 2 的页面做到一半时开始（需要有页面可接入）
- 商户号申请是异步等待，不阻塞开发
