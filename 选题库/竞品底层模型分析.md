# 竞品底层模型分析

> 基于 2026-03-16 收集的 80+ 个成功小程序，提炼底层选题思路和商业模型。

---

## 一、核心发现：这些小程序在做同一件事

表面上看，这些小程序涵盖接龙、签到、班级管理、工时记录等十几个品类。但剥开功能层，它们在做**同一件事**：

> **把微信群里本来靠"刷屏聊天"完成的协作行为，变成一个结构化的工具。**

- 群里说"报名的接龙"→ 接龙工具
- 群里说"到了的打个1"→ 签到工具
- 群里说"作业拍照发群里"→ 作业收集工具
- 群里说"同意的举手"→ 投票工具
- 群里说"这个月班费收支"→ 班费管理工具

**这不是偶然的。微信群是中国最大的非正式组织单元，但群聊天的信息流天生不适合做"协作"——消息会被刷走、数据无法汇总、结果无法追溯。** 这些小程序填的就是这个结构性缺口。

---

## 二、5 种底层选题模型

### 模型 A：群管理者工具（占比最大，约 60%）

**典型代表**：接龙、接龙管家、群报数、秒应、群投票、群收集、群打卡、群签到助手

**底层逻辑**：
```
用户不是"每个群成员"，而是"群管理者"
          │
          ▼
群管理者有反复出现的协作需求
（收集信息、统计结果、通知确认、组织活动）
          │
          ▼
工具被群管理者分享到群里 → 全员被动使用
          │
          ▼
群管理者感受到效率提升 → 愿意为高级功能付费
```

**关键特征**：
| 维度 | 分析 |
|------|------|
| **真正的用户** | 不是群成员，是群管理者（老师/班委/HR/团长/社区负责人） |
| **获客方式** | 管理者发到群里 → 全群被动激活 → 1 个付费用户带来 30-500 个免费用户 |
| **频率** | 管理者：每周多次；成员：被动响应 |
| **付费动机** | 管理者为"省时间+显得专业+数据可追溯"付费 |
| **竞争壁垒** | 几乎没有功能壁垒，壁垒是 SEO 占位 + 用户习惯 |

**为什么这个模型能跑通**：
1. **天然病毒传播**：工具本身必须在群里分享才能使用，每次使用就是一次传播
2. **管理者有支付动机**：他们的痛点是"管理 50-500 人的协作"，省时间值钱
3. **切换成本低但习惯成本高**：功能同质化严重，但"这个群一直用 A 工具"的惯性很强
4. **赢家通吃的 SEO 战争**：用户搜"接龙"只看前 3 个结果，所以铺 10 个关键词的公司比做 1 个好产品的个人开发者更成功

### 模型 B：班级/教育垂直生态（约 20%）

**典型代表**：班小二（9个子产品）、云班级、作业助手、班级喵、群收作业、1530小助手

**底层逻辑**：
```
班级 = 中国最标准化的"群组织"
  │
  ├── 固定人群（老师+家长+学生，30-50人/班）
  ├── 固定节奏（每天有作业，每周有通知，每学期有考试）
  ├── 强制参与（家长不能不配合老师）
  └── 情绪加持（家长对孩子教育焦虑 → 付费意愿极高）
          │
          ▼
从一个刚需切入 → 横向扩展覆盖班级全场景
（作业→成绩→通知→打卡→缴费→采购→打印）
```

**关键特征**：
| 维度 | 分析 |
|------|------|
| **真正的用户** | 老师 + 班级家委 |
| **获客方式** | 老师在家长群推荐 → 全班家长必须用（无法拒绝） |
| **频率** | 极高——每天都有作业/通知/打卡 |
| **付费动机** | 学校付费（校园版）+ 家长为孩子学习付费（智学/打印） |
| **竞争壁垒** | 生态锁定——一旦班级用了你的系统，切换成本极高 |

**班小二的生态扩展路径（值得学习）**：
```
班小二（免费班级管理）
  ├── 校园班小二（学校付费版 → B端收入）
  ├── 班小二Pro（PC端 → 提高留存）
  ├── 班小二智校（给校长/园长 → 决策者影响）
  ├── 小二智学（家长课后辅导 → C端付费）
  ├── 班小二家长慧（家庭教育 → 内容付费）
  ├── 班小二云打印（打印服务 → 电商变现）
  ├── 班小二店铺（学习用品 → 电商变现）
  └── 班小二知途印迹（学生笔记打卡 → 新场景）
```

**核心洞察**：班小二不是做了 9 个产品，而是**用一个免费工具占领了"班级"这个组织节点，然后围绕这个节点的所有消费场景变现**。

### 模型 C：蓝领/倒班人群工具（约 5%）

**典型代表**：小时工记工时、安心记加班、安心排班日历、云人事

**底层逻辑**：
```
蓝领工人的核心焦虑：我的工时有没有被少算？
          │
          ▼
自己记一份 → 和老板/公司的对比 → 保护自己的利益
          │
          ▼
每天都要记 → 高频刚需 → 形成习惯
          │
          ▼
免费用 → 高级功能（工资计算、导出、多工种）→ 付费
```

**关键特征**：
| 维度 | 分析 |
|------|------|
| **真正的用户** | 时薪制/倒班制工人（服务员、工厂工人、护士、保安等） |
| **获客方式** | 搜索"记工时""记加班" → 个人下载使用 |
| **频率** | 每天 |
| **付费动机** | 直接关系到钱——记准工时 = 拿对工资 |
| **情感驱动** | 自我保护（"公司少算我工时怎么办"）+ 掌控感 |

**上海慧贤的升级路径**：
```
安心记加班（个人记录）→ 安心排班日历（排班管理）→ 云人事（企业HR系统）
       C端免费                C端付费              B端付费
```

**核心洞察**：从个人的"自我保护"需求切入，逐步升级到团队/企业管理工具。用户从 C 端个人长成 B 端决策者。

---

### 模型 D：习惯养成 + 生活方式（约 5%）

**典型代表**：小来早晚安（1000万用户）、小来时光、小来生活馆

**底层逻辑**：
```
打卡 = 最简单的习惯养成行为
  │
  ├── 早安/晚安打卡 → 每天 2 次，极高频
  ├── 精美打卡图 → 朋友圈分享 → 社交货币 → 自传播
  └── 励志金句 → 情绪价值 → 留存
          │
          ▼
1000万用户的注意力池
          │
          ▼
小来时光（纪念日提醒）→ 日常触达
小来生活馆（精选电商）→ 流量变现
```

**关键特征**：
| 维度 | 分析 |
|------|------|
| **真正的用户** | 25-45岁女性，追求自律/精致生活 |
| **获客方式** | 用户主动分享打卡图到朋友圈 → 好友看到 → 下载 |
| **频率** | 每天 2 次（早安+晚安） |
| **付费动机** | 前期不付费；规模化后通过电商变现 |
| **情感驱动** | 成就感 + 社交认同 + 自律感 |

**核心洞察**：不靠工具本身赚钱，靠工具积累的用户规模做电商/广告变现。**这个模型需要先到千万级用户才有意义，不适合一个人做。**

---

### 模型 E：个人开发者的"关键词占位"（约 10%）

**典型代表**：扫码签到系列（5个近似小程序）、解析去水印助手

**底层逻辑**：
```
找到一个搜索量大的关键词
  │
  ▼
做最简功能（MVP 极简）
  │
  ▼
靠关键词排名获取自然流量
  │
  ▼
广告变现 或 付费解锁
```

**扫码签到系列的 SEO 矩阵**：
```
同一个开发者做了 5 个功能几乎一样的小程序：
  扫码签到     → 关键词"扫码签到"
  扫码登记预约  → 关键词"扫码登记""扫码预约"
  扫码打卡工具  → 关键词"扫码打卡"
  二维码签到工具 → 关键词"二维码签到"
  扫码预约     → 关键词"扫码预约""二维码报名"
```

**核心洞察**：不是做 5 个产品，是用 5 个名字覆盖 10+ 个搜索关键词。**产品就是关键词本身。**

## 三、跨模型的共性规律

### 规律 1：成功的小程序都在服务"组织者"而非"参与者"

| 小程序 | 组织者 | 参与者 |
|--------|--------|--------|
| 接龙 | 发起接龙的人（老师/群主） | 跟着接龙的人 |
| 签到 | 组织活动的人 | 来签到的人 |
| 投票 | 发起投票的人 | 投票的人 |
| 班级管理 | 老师/班委 | 家长/学生 |

**付费的永远是组织者，参与者免费。** 因为组织者有管理压力、有效率诉求、有"显得专业"的需求。

### 规律 2：获客靠"被迫使用"而非"主动下载"

这些小程序最强的增长飞轮不是广告，是：

```
组织者在群里发了一个接龙/签到/投票
    → 群里所有人必须点进去操作
    → 操作时看到品牌和功能
    → 其中一些人自己也是其他群的组织者
    → 他们也开始用这个工具
    → 循环
```

**一个付费用户（组织者）= 一群免费用户（参与者）= 新的潜在付费用户。**

### 规律 3：变现靠"管理者升级"而非"内容付费"

主流变现路径：

```
免费版（核心功能完整）
  │
  ├── VIP 月卡/年卡（19.9-99/年）
  │     ├── 人数上限提升（50→无限）
  │     ├── 数据导出 Excel
  │     ├── 自定义品牌（去除工具水印）
  │     ├── 高级统计/分析
  │     └── 去广告
  │
  └── 场景延伸付费
        ├── 打印（云打印）
        ├── 电商（学习用品）
        └── 企业版/学校版（B端定价）
```

### 规律 4：壁垒不在功能，在占位

这些小程序的功能**极度同质化**——接龙管家、群报数、秒应、接龙+……做的事几乎一样。那为什么能共存？

```
壁垒来源：
  ① SEO 占位 — 搜"接龙"出现在第 1 位 vs 第 10 位，流量差 100 倍
  ② 用户习惯 — "我们班一直用这个"
  ③ 矩阵效应 — 10 个小程序 × 每个 3 个关键词 = 30 个入口
  ④ 关联企业 — 用多个公司主体规避平台对同一主体的限制
```

### 规律 5：个人开发者的生存空间在"长尾关键词"

| 公司型玩家 | 个人开发者 |
|-----------|-----------|
| 做"接龙"这种大词 | 做"小时工记工时""去水印"这种长尾词 |
| 10+ 小程序矩阵 | 1-5 个小程序 |
| VIP 订阅制 | 广告 + 简单付费 |
| 团队维护 | 做完基本不更新 |
| 获客靠矩阵 SEO | 获客靠精准关键词 |

## 四、对"小程序工厂"选题的启示

### 这些成功小程序 vs 我们的选题标准

| 我们的标准 | 这些小程序符合吗？ | 意味着什么 |
|-----------|------------------|-----------|
| 1. 适合一个人起步 | 大部分最初是一个人/小团队 | **符合** |
| 2. 依赖大模型实现本质突破 | **完全不依赖 AI** | 这是我们的差异化机会 |
| 3. 高频复购行为场景 | 是，每天/每周使用 | **符合** |
| 4. 用户画像清晰 | 是，目标非常明确 | **符合** |
| 5. 点子具体可执行 | 是，功能很聚焦 | **符合** |
| 6. 消费群体足够大 | 是，百万到亿级 | **符合** |
| 7. 基于真实高频痛点 | 是，替代了群聊的低效协作 | **符合** |
| 8. 符合人性深层驱动 | 部分符合（责任感、掌控感） | **部分符合** |

### 关键结论

**这些小程序验证了 8 项标准中的 6 项都很重要。但它们完全不涉及标准 2（大模型）。这恰好意味着：**

> 在这些已验证的高频场景中，找到"AI 能带来 10 倍体验提升"的环节，就是我们的机会。

### 具体方向思考

| 已验证的场景 | AI 可以做什么 | 原来做不到/成本极高 |
|-------------|-------------|-------------------|
| 作业批改（班小二/作业助手） | AI 逐题批改+生成反馈 | 老师人工批改 50 份要 2 小时 |
| 成绩查询（来豹查查/易查成绩） | AI 生成个性化学习建议 | 每个学生的薄弱点分析需要专业教师 |
| 工时记录（小时工记工时） | AI 自动识别排班模式+预测收入 | 蓝领不会用 Excel 分析 |
| 通知/公告（通知小工具） | AI 自动生成通知文案 | 组织者每次要自己想怎么写 |
| 投票/问卷（群投票/群问卷） | AI 自动分析投票结果+生成结论 | 结果出来了但没人会分析 |
| 睡前故事（睡前故事阅读） | AI 个性化生成故事（加入孩子名字/喜好） | 故事库有限，千篇一律 |
| 奖状管家 | AI 根据场景自动设计奖状 | 模板有限，自己设计需要设计能力 |

### 不该做的方向

| 方向 | 原因 |
|------|------|
| 又一个接龙工具 | 红海中的红海，没有 AI 差异化空间 |
| 纯签到/打卡工具 | 功能太简单，AI 无法增值 |
| 纯工具型小程序（去水印等） | 不符合标准 2（不依赖大模型），且容易被抄 |
| 需要千万用户才能变现的模型（小来早晚安） | 不符合标准 1（一个人做不到那个规模的增长） |

## 五、模型图谱总结

```
成功小程序的底层公式：

  高频群协作场景
    × 服务组织者（而非参与者）
    × 天然群传播（使用即推广）
    × 免费核心功能 + VIP 管理升级
    × SEO 关键词占位

我们的叠加变量：

  + AI 驱动的本质体验提升
    （不是加个"AI 问答"，而是 AI 替代了原来做不到的事）
```
