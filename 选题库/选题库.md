## 竞品模式总结

### 典型打法

1. **矩阵铺量**：一家公司注册多个关联小程序覆盖不同关键词（肥迪13个、快云通合22个、接龙管家4个）
2. **免费+VIP**：基础功能免费，高级功能/去广告用VIP（接龙VIP、接龙管家VIP、群报数会员版）
3. **场景延伸**：从核心场景向周边扩展（班小二：班级管理→智学→云打印→店铺；慧贤：记加班→排班→人事管理）
4. **关联企业**：用多个公司主体注册同一产品的不同版本，规避平台限制

### 赛道拥挤度

| 赛道 | 拥挤度 | 代表玩家 |
|------|--------|---------|
| 群接龙/数据收集 | 极高 | 接龙、接龙管家、群报数、秒应 |
| 签到打卡 | 高 | 打卡签到助手、扫码签到系列 |
| 班级管理 | 高 | 班小二、云班级、班级喵 |
| 成绩查询 | 中 | 来豹查查、易查成绩 |
| 工时/考勤 | 中 | 小时工记工时、安心记加班 |
| 投票/抽签 | 中 | 群投票、抽签帮手 |
| 相册/生活 | 低 | 秀秀群相册、小来系列 |
| 去水印 | 低 | 解析去水印助手 |

### 对我们的启示

- **红海场景避开**：群接龙、签到打卡、班级管理已被充分覆盖，不适合新进入
- **矩阵策略可借鉴**：但需要结合我们的选题标准（标准2：依赖大模型实现本质突破），而非简单复制
- **关注"有流量但没AI化"的场景**：如工时记录、排班管理、成绩分析——目前都是纯工具，AI 可以带来差异化
- **个人开发者能活**：小时工记工时、解析去水印助手、扫码签到系列都是个人开发者作品，说明小程序仍有个人开发者空间
