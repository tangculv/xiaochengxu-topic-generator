# 选题生成器 — 底层结构与组合机制

> 上一版的问题：列了 6 个选题，那是"想法"，不是结构。
> 这一版要做的：找到能长出几百几千个选题的生成性结构。

---

## 零、最深的一层：这些小程序到底在做什么

不是"工具"。不是"效率提升"。更深的一层：

> **每一个成功的小程序，都是把一个"反复出现的社会义务"变成了一次结构化的数字交互。**

什么叫"社会义务"？

- 老师在群里说"明天春游，报名接龙"——你作为家长，**有义务回复**
- 领导说"到了的签个到"——你**有义务签**
- 家委说"这学期班费，每人 200"——你**有义务交**
- 你自己打了 8 小时工——你**有义务（对自己）记下来**，否则月底说不清

接龙工具不是"帮你接龙"。它是**帮你履行一个社会义务，并让这个履行过程变得可追溯**。

签到工具不是"帮你签到"。它是**帮组织者确认每个人是否履行了到场的义务**。

班小二不是"班级管理"。它是**把老师-家长之间几十种反复出现的义务（交作业、看通知、交费、打卡）打包成结构化流程**。

**义务 = 有人必须做某件事 + 有人必须确认它被做了 + 这件事会反复出现。**

这三个条件同时满足，就是一个小程序的种子。

## 一、生成结构：4 个独立维度

一个选题 = 4 个维度的组合。维度之间互相独立，可以自由排列。

```
选题 = 义务关系 × 结构化动作 × 作用领域 × AI 能力层
```

### 维度 A：义务关系（谁对谁有义务？）

义务不是凭空产生的，它来自**关系**。关系决定了谁是组织者、谁是参与者、义务有多强。

| 关系类型 | 义务方向 | 义务强度 | 典型角色 |
|---------|---------|---------|---------|
| **管教型** | 上→下 指派，下→上 服从 | 极强（不做会有后果） | 老师→家长/学生，领导→下属，教练→学员 |
| **照护型** | 强→弱 保护，弱→强 依赖 | 极强（情感+责任） | 父母→孩子，子女→老人，医生→患者 |
| **契约型** | 双向 约定义务 | 强（涉及利益） | 雇主↔雇员，商家↔客户，房东↔租户 |
| **互助型** | 平等成员间互相协调 | 中（社交压力） | 邻居↔邻居，同事↔同事，同学↔同学 |
| **自律型** | 自己→自己 | 弱到强（看动机） | 个人健康管理，个人学习，个人记账 |

**这个维度为什么重要**：义务强度决定了用户"不得不用"的程度。接龙之所以成功，不是因为好用，是因为老师发了你就**必须**回。义务越强，获客越不需要营销。

**可枚举的关系**（不完全列举，每类可继续展开）：

管教型：
- 班主任 → 家长
- 科目老师 → 学生
- 辅导班老师 → 学员/家长
- 教练 → 队员
- 部门主管 → 员工
- 项目经理 → 团队成员
- 社区干部 → 居民
- 宗教领袖 → 信众
- 班委/家委 → 本班家长

照护型：
- 父母 → 婴幼儿
- 父母 → 学龄儿童
- 父母 → 青春期孩子
- 成年子女 → 老年父母
- 家属 → 慢病患者
- 家属 → 术后恢复者
- 家属 → 孕产妇
- 宠物主 → 宠物

契约型：
- 雇主 → 小时工/兼职
- 雇主 → 全职员工
- 团长 → 团购顾客
- 房东 → 租户
- 物业 → 业主
- 培训机构 → 付费学员
- 自由职业者 → 客户

互助型：
- 班级家长 ↔ 班级家长
- 同小区业主 ↔ 业主
- 兴趣社群成员 ↔ 成员
- 同事 ↔ 同事
- 病友 ↔ 病友
- 家族成员 ↔ 家族成员

自律型：
- 自己管理体重/饮食
- 自己管理工时/收入
- 自己管理学习进度
- 自己管理情绪/睡眠
- 自己管理财务

### 维度 B：结构化动作（把什么混沌行为变成结构？）

群聊里的协作行为本来是混沌的（消息流、刷屏、信息淹没）。小程序的核心动作是**结构化**。

但"结构化"不是一个动作，而是 7 种基础动作：

| 动作 | 含义 | 信息流向 | 竞品对应 |
|------|------|---------|---------|
| **收集** | 从多人那里聚合信息 | 多→一 | 接龙、报名、收作业、问卷 |
| **分发** | 向多人推送信息 | 一→多 | 通知、发成绩、发任务 |
| **核验** | 确认某人是否完成了某事 | 一↔一 | 签到、打卡、考勤 |
| **评判** | 对收集到的内容做质量判断 | 一→一 | 批改、评分、审核 |
| **决策** | 在多个选项中做出选择 | 多→一 | 投票、抽签、抽奖 |
| **追踪** | 记录状态随时间的变化 | 一→时间线 | 记工时、记账、成长记录 |
| **协调** | 对齐多人的时间/资源 | 多↔多 | 排班、预约、调度 |

**关键洞察**：现有竞品几乎只做了"收集"和"核验"，少量做"分发"和"决策"。**"评判"和"追踪+分析"两个动作基本是空白**——因为这两个动作需要**判断力**，以前只有人能做。

这就是 AI 切入的结构性位置。

### 维度 D：AI 能力层（AI 在哪个环节创造"以前做不到"的价值？）

| AI 能力 | 做了什么 | 替代了谁 |
|--------|---------|---------|
| **理解非结构化输入** | 看懂手写、拍照、语音、视频中的内容 | 人工逐个检查的人 |
| **质量评判** | 判断对错、好坏、合格与否 | 专业评审者（老师、医生、教练） |
| **个性化生成** | 为每个人生成不同的反馈/建议/内容 | 一对一的专家服务 |
| **模式识别** | 从历史数据中发现趋势、异常、规律 | 数据分析师 |
| **知识推理** | 基于专业知识给出判断和建议 | 领域专家（营养师、理财师、教育顾问） |
| **内容创作** | 写文案、生成图片、编排内容 | 文案写手、设计师 |
| **格式转换** | 在不同表达方式之间转换 | 翻译、排版、整理的人 |

## 二、组合规则：什么组合是有效的

不是所有 A×B×C×D 的组合都成立。有效组合需要满足几个约束：

### 约束 1：义务必须是真实存在且反复出现的

```
✅ 老师要求家长每天检查作业签字 → 真实义务，每天出现
✅ 小时工需要记录每天工时 → 真实义务，每天出现
❌ 邻居之间互相评价厨艺 → 不是义务，不会反复出现
```

**判断方法**：这个义务如果不履行，会有什么后果？后果越严重、越确定，义务越真实。

### 约束 2：结构化动作必须比现有方式好 10 倍

```
✅ 50 人接龙报名：群聊要刷 50 条消息 → 工具一个链接搞定 → 10倍提升
✅ AI 批改 50 份作业：老师要花 2 小时 → AI 10秒 → 100倍提升
❌ 3 个人约吃饭时间：微信里商量一下就行 → 不需要工具
```

**判断方法**：这件事现在是怎么做的？用了工具后体验提升了多少？不到 10 倍别做。

### 约束 3：AI 层必须是"做不到→做得到"而非"慢→快"

```
✅ 看懂手写数学题并判断对错 → 以前家长做不到（不会数学）
✅ 分析 3 个月饮食数据给出营养建议 → 以前要请营养师
❌ 用 AI 自动发通知 → 以前人也能发，只是手动 → 这不是本质突破
```

**判断方法**：如果去掉 AI，用户花钱花时间能自己做吗？如果能，AI 只是锦上添花，不是本质突破。

### 约束 4：传播机制必须内嵌于使用行为

```
✅ 接龙：发到群里才能用 → 使用=传播
✅ 签到：组织者分享到群里 → 全员激活
✅ 打卡教练：打卡结果发在群里 → 其他群主看到也想用
❌ 个人记账：自己用 → 没有传播机制
```

自律型关系（维度 A 第 5 类）天然缺乏传播机制，需要额外设计（比如成果分享、排行榜、病友群推荐）。

## 三、生成示范：从一个关系节点长出一棵树

以 **"成年子女 → 老年父母"** 这个关系为例，展示如何从一个节点系统性地长出选题：

### 第一层：这个关系中有哪些反复出现的义务？

```
成年子女对老年父母的义务：
  ├── 健康监护：关心身体状况、用药、饮食
  ├── 就医协助：陪诊、记录医嘱、管理检查结果
  ├── 情感联络：保持联系、节日关怀、陪伴
  ├── 安全确认：知道父母是否安全、有没有摔倒
  ├── 财务协助：帮处理缴费、养老金、保险
  └── 生活协调：兄弟姐妹间分工谁来做什么
```

### 第二层：每个义务可以做哪些结构化动作？

以"健康监护"为例：

```
健康监护 ×
  ├── 收集：收集父母每天的血压/血糖/饮食数据
  ├── 追踪：记录数据变化趋势
  ├── 核验：确认父母是否按时吃药
  ├── 评判：判断数据是否异常、饮食是否合理
  ├── 分发：把分析结果分享给其他兄弟姐妹
  ├── 协调：协调谁这周陪父母去复查
  └── 决策：多个治疗方案该选哪个
```

### 第三层：AI 能在哪些动作上创造本质价值？

```
收集 + AI理解：父母拍一下吃的东西 → AI 自动识别食物和份量 → 不需要手动输入
追踪 + AI模式识别：3 个月血压数据 → AI 发现"每周一血压偏高" → 人看不出这个规律
核验 + AI理解：父母拍药盒 → AI 确认今天的药吃了没、剩多少 → 远程确认
评判 + AI知识推理：今天吃了红烧肉+啤酒 → AI 结合痛风病史说"尿酸会升高" → 替代营养师
分发 + AI生成：自动生成"本周父母健康周报"发给兄弟姐妹 → 替代子女手动汇报
```

### 第四层：每个有效组合就是一个选题种子

仅"成年子女→老年父母 × 健康监护"这一条线，就生出至少 5 个选题种子。

再加上其他义务线（就医、安全、生活协调），这个关系节点能生出 **15-20 个选题种子**。

维度 A 中列了 **30+ 个关系**，每个关系能生出 10-20 个种子。

**30 × 15 = 450 个选题种子。**

用选题标准过滤（频率不够高的、人群不够大的、AI 无法突破的），保守估计留下 20%-30%。

**≈ 100-150 个值得评估的选题。**

## 四、再做一棵树：从"培训班老师 → 学员"长出来

### 义务分解

```
培训班老师对学员的义务：
  ├── 教学：上课、讲解
  ├── 布置练习：课后练习、作业
  ├── 评估反馈：检查练习质量、给反馈
  ├── 进度追踪：记录每个学员的进步
  ├── 沟通：和家长/学员沟通学习状况
  └── 管理：排课、考勤、收费
```

### 动作 × 领域展开（以"评估反馈"为例）

```
评估反馈 × 技能领域：
  ├── 钢琴练习 → 收集录音 → AI 评判音准/节奏/表现力 → 生成个性化练习建议
  ├── 书法练字 → 收集照片 → AI 评判笔画/结构/章法 → 标注问题并示范
  ├── 英语口语 → 收集录音 → AI 评判发音/流利/语法 → 生成纠正建议
  ├── 美术绘画 → 收集照片 → AI 评判构图/色彩/技法 → 对比名作分析
  ├── 体育动作 → 收集视频 → AI 评判姿势/节奏/力度 → 标注问题关节
  ├── 编程作业 → 收集代码 → AI 评判正确性/风格/效率 → 生成改进建议
  └── 舞蹈动作 → 收集视频 → AI 评判节奏/姿态/表情 → 逐拍分析
```

**仅"培训班老师 × 评估反馈"这一个交叉点，乘以不同技能领域，就是 7 个选题。** 加上其他义务线，这个关系节点又能长出 20-30 个。

## 五、维度 B 的深层展开：7 种动作的子类型

上面的"7 种动作"还不够细。每种动作内部还有子类型，它们产生不同的产品形态：

### 收集的子类型

| 子类型 | 收集什么 | 产品形态 | 例子 |
|--------|---------|---------|------|
| 收意向 | 是/否/选项 | 接龙/投票 | 接龙、报名 |
| 收提交物 | 文件/照片/视频/文字 | 提交+归档 | 收作业、收截图 |
| 收数据 | 数值/打卡/记录 | 表单+统计 | 收体温、收血压 |
| 收款项 | 钱 | 收款+对账 | 收班费、收活动费 |
| 收反馈 | 评价/意见/建议 | 问卷+分析 | 满意度、需求收集 |

### 评判的子类型

| 子类型 | 评判什么 | 需要的判断力 | AI 价值 |
|--------|---------|-----------|---------|
| 判对错 | 答案正确性 | 学科知识 | 极高（替代老师批改） |
| 判质量 | 作品/练习好坏 | 专业审美/技术标准 | 高（替代教练点评） |
| 判合规 | 是否符合标准/规范 | 规则理解 | 高（替代审核员） |
| 判健康 | 数据是否正常 | 医学知识 | 极高（替代医生初判） |
| 判风险 | 有没有问题/隐患 | 经验判断 | 高（替代专家预警） |

### 追踪的子类型

| 子类型 | 追踪什么 | 时间尺度 | AI 价值 |
|--------|---------|---------|---------|
| 追踪习惯 | 每天是否完成 | 天→月 | 中（发现坚持/放弃模式） |
| 追踪进步 | 能力/水平变化 | 周→年 | 高（量化不可见的进步） |
| 追踪健康 | 体征/症状变化 | 天→年 | 极高（发现趋势和异常） |
| 追踪财务 | 收入/支出变化 | 天→月 | 中（分析消费模式） |
| 追踪成长 | 孩子的发育/学习里程碑 | 月→年 | 高（对比发育标准） |

## 六、AI 能力层的深层展开：从"有/无 AI"到"AI 做了什么级别的事"

不是所有 AI 应用价值相同。分三级：

| 级别 | AI 做的事 | 替代的人 | 用户感知 | 付费意愿 |
|------|---------|---------|---------|---------|
| **L1 替代重复劳动** | 自动分类、格式转换、定时提醒 | 文员/助理 | "方便了" | 低（9.9/月） |
| **L2 替代技能工作** | 批改、评估、写文案、分析数据 | 教师/文案/分析师 | "以前做不了，现在能做了" | 中（19.9-39.9/月） |
| **L3 替代专家判断** | 医学建议、教育诊断、风险预警 | 医生/咨询师/专家 | "相当于请了个私人顾问" | 高（39.9-99/月） |

**我们的选题标准要求 L2 以上。** L1 没有本质突破，容易被抄，不值得做。

## 七、生成器使用方法

### 步骤 1：选一个关系（维度 A）

从关系列表中选一个。优先选：
- 义务强度高的（管教型、照护型 > 互助型 > 自律型）
- 涉及人群规模大的

### 步骤 2：展开这个关系中的义务

问：这个关系中，哪些事情是必须做的、反复出现的？列出所有义务。

### 步骤 3：每个义务 × 7 种动作

对每个义务，过一遍 7 种动作（收集、分发、核验、评判、追踪、协调、决策），看哪些组合是真实场景。

### 步骤 4：对每个有效组合，问 AI 能做什么

对每个"关系 × 义务 × 动作"的组合，问：
- 这个动作中，哪个环节以前需要专家/大量人力？
- AI 能做到 L2 以上吗？

### 步骤 5：用选题标准过滤

对通过上述步骤的选题种子，用 8 项标准打分，≥70 分进入候选池。

### 输出格式

```
关系：[谁 → 谁]
义务：[什么义务]
动作：[什么结构化动作]
领域：[关于什么事]
AI 层级：[L1/L2/L3 + 具体做什么]
一句话描述：[用户做什么 → 得到什么]
```

## 八、快速验证清单

一个组合要成为选题，必须通过以下 5 个问题：

```
1. 义务真实吗？
   → 不做会怎样？（越有后果越好）

2. 频率够高吗？
   → 每天？每周？每月？（低于每周的谨慎考虑）

3. 现在怎么解决的？
   → 如果答案是"凑合/忍着/花大钱"，机会大
   → 如果答案是"已有好工具"，放弃

4. AI 做到了什么以前做不到的事？
   → 必须是 L2 以上
   → 必须是"做不到→做得到"，不是"慢→快"

5. 谁付钱？为什么这个钱值得花？
   → 具体到人（不是"用户"）
   → 具体到价格（不是"有付费意愿"）
```

## 九、结构的力量：为什么这套框架是可靠的

回到最初的问题：为什么底层原理一致就能保证新选题也能成？

```
接龙 = 管教型关系(老师→家长) × 收集(收意向) × 活动 × 无AI
签到 = 管教型关系(组织者→参与者) × 核验(出勤) × 时间 × 无AI
班小二 = 管教型关系(老师→家长) × [收集+分发+核验+追踪] × 学业 × 无AI
记工时 = 契约型关系(雇员→雇主) × 追踪(工时) × 金钱 × 无AI
```

它们成功，不是因为"接龙"这个功能好，而是因为：
- 关系产生了真实的、不可回避的义务
- 义务在当前方式下履行起来很痛苦
- 工具让履行变得轻松
- 工具的使用方式天然就在群里传播

**这 4 个条件是结构性的，不依赖于具体品类。** 任何满足这 4 个条件的组合，都有相同的成功基础。

我们在这个基础上加了第 5 个条件：
- **AI 让工具能做到"以前只有专家才能做的事"**

这第 5 个条件不改变底层结构，只改变价值层级。所以生成出来的选题，**底座和已验证的成功产品完全一致，上层多了一个差异化壁垒**。
