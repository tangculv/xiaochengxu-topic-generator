# 小程序自动选题与评估体系

## 系统目标

- **自动产生选题**：从多个来源持续收集候选选题
- **自动评估选题**：按照8大核心标准量化评分
- **轮训执行**：定期运行，保持选题库更新
- **智能推荐**：输出高分选题供人工确认

---

## 整体架构

```
选题源采集 → 选题库 → 自动评估 → 排序筛选 → 候选选题 → 人工确认
    ↑                                                     ↓
    └────────────────────── 反馈优化 ←────────────────────┘
```

## 1. 选题来源设计

### 1.1 来源类型

| 来源 | 描述 | 数据格式 | 更新频率 |
|------|------|---------|---------|
| **研究报告提取** | 从2025中国数字社会小程序机遇研究报告提取7个选题 | Markdown | 手动触发 |
| **热点挖掘** | 从社交平台/搜索引擎挖掘热点 | API/爬虫 | 每周 |
| **AI生成** | 用AI生成新点子 | Prompt → 输出 | 每天 |
| **竞品分析** | 分析竞品产品，发现差异化机会 | 竞品列表 | 每周 |
| **用户需求** | 从用户反馈/社区需求中提取 | 反馈列表 | 实时 |

### 1.2 选题数据结构

每个选题包含以下字段：

```yaml

source: "来源"
created_at: "2026-03-06"

# 评估数据
evaluation:
  total_score: 0
  standard_1: 0
  standard_2: 0
  # ... standard_8: 0

# 评估备注
notes: ""

```

## 2. 自动评估系统

### 2.1 评估标准（8大核心）

| 标准编号 | 标准名称 | 满分 | 自动化评估指标 |
|---------|---------|------|-------------|
| S1 | 适合一个人起步 | 10 | 技术复杂度、开发周期、资源需求 |
| S2 | 依赖大模型实现本质突破 | 15 | AI角色、降本倍数、能力提升 |
| S3 | 高频复购行为场景 | 10 | 使用频率、复购动机、付费设计 |
| S4 | 用户画像清晰，支付动机明确 | 15 | 用户画像、付费动机、价格周期 |
| S5 | 点子具体、可执行 | 10 | 使用场景、产品形态、启动路径 |
| S6 | 消费群体足够大 | 10 | 目标用户规模、市场可扩展性 |
| S7 | 基于真实、高频痛点 | 15 | 痛点真实性、问题频率、解决成本 |
| S8 | 符合人性深层驱动 | 15 | 人性驱动类型、情感强度 |

### 2.2 评估指标量化

每个标准有明确的评分规则：

#### S1: 适合一个人起步（10分）
- 技术复杂度：5分（简单） / 3分（中等） / 1分（复杂）
- 开发周期：3分（≤4周） / 2分（4-8周） / 1分（>8周）
- 资源需求：2分（低） / 1分（中等） / 0分（高）

#### S2: 依赖大模型实现本质突破（15分）
- AI角色：8分（核心功能） / 4分（辅助功能） / 0分（无）
- 降本倍数：4分（≥10倍） / 2分（5-10倍） / 1分（<5倍）
- 能力提升：3分（实现不可能） / 2分（显著提升） / 1分（轻微提升）

#### S3: 高频复购行为场景（10分）
- 使用频率：4分（每天） / 3分（每周） / 2分（每月） / 1分（偶尔）
- 复购动机：3分（强） / 2分（中） / 1分（弱）
- 付费设计：3分（明确） / 1分（模糊） / 0分（无）

#### S4: 用户画像清晰，支付动机明确（15分）
- 用户画像：5分（4类以上） / 3分（2-3类） / 1分（1类）
- 付费动机：5分（明确且强） / 3分（明确） / 1分（模糊）
- 价格周期：5分（明确） / 3分（部分明确） / 1分（无）

#### S5: 点子具体、可执行（10分）
- 使用场景：3分（清晰） / 1分（一般） / 0分（模糊）
- 产品形态：3分（明确） / 1分（一般） / 0分（模糊）
- 启动路径：4分（明确3步以上） / 2分（明确2步） / 1分（模糊）

#### S6: 消费群体足够大（10分）
- 目标用户规模：8分（>1000万） / 5分（100-1000万） / 2分（10-100万） / 0分（<10万）
- 市场可扩展性：2分（高） / 1分（中） / 0分（低）

#### S7: 基于真实、高频痛点（15分）
- 痛点真实性：5分（验证过） / 3分（合理推断） / 1分（假设）
- 问题频率：5分（每天） / 3分（每周） / 1分（每月）
- 解决成本：5分（高） / 3分（中） / 1分（低）

#### S8: 符合人性深层驱动（15分）
- 人性驱动类型：5分（强驱动） / 3分（中驱动） / 1分（弱驱动）
- 情感强度：5分（强） / 3分（中） / 1分（弱）
- 复购潜力：5分（高） / 3分（中） / 1分（低）

### 2.3 评估流程

1. **输入**：选题基本信息（标题、描述、目标用户等）
2. **AI分析**：根据选题描述，量化8大标准每个指标
3. **自动评分**：按照评分规则计算总分
4. **生成报告**：输出详细评估报告和改进建议
5. **存档**：存入选题库，标记为"evaluated"

## 3. 轮训执行机制

### 3.1 轮训任务

| 任务 | 触发条件 | 执行频率 | 输出 |
|------|---------|---------|------|
| **采集新选题** | 定时触发 | 每天1次 | 新选题列表 |
| **自动评估选题** | 有新选题 | 自动触发 | 评估报告 |
| **生成候选推荐** | 定时触发 | 每周1次 | 高分选题列表 |
| **优化评估模型** | 人工反馈 | 按需 | 评分规则更新 |

### 3.2 执行流程

```
每天 09:00 → 采集新选题（AI生成+热点挖掘）
每天 10:00 → 自动评估新选题
每天 11:00 → 更新选题库排序
每周日 → 生成候选推荐列表
```

### 3.3 人工干预点

| 环节 | 人工操作 | 触发条件 |
|------|---------|---------|
| 选题采集 | 确认/拒绝AI生成的选题 | AI生成后 |
| 评估结果 | 调整评分/补充信息 | 评估报告生成后 |
| 选题决策 | 确认/拒绝候选选题 | 查看候选推荐后 |
| 评估优化 | 更新评分规则 | 发现评估不准确时 |

## 6. 执行脚本设计

### 6.1 选题采集脚本（collect-topics.py）

功能：
- 从研究报告提取选题
- AI生成新选题
- 热点挖掘（API接入）
- 竞品分析
- 存入 `topic-library/pending/`

### 6.2 自动评估脚本（auto-evaluate.py）

功能：
- 扫描 `topic-library/pending/`
- 调用AI进行评估
- 按照评分规则计算总分
- 生成评估报告到 `evaluation-reports/`
- 移动到 `topic-library/evaluated/`

### 6.3 候选推荐脚本（generate-candidates.py）

功能：
- 扫描 `topic-library/evaluated/`
- 按总分排序
- 过滤高分选题（≥80分）
- 生成候选推荐列表到 `candidate-recommendations/`

### 6.4 轮训执行脚本（run-pipeline.sh）

功能：
- 按定时任务执行整个流程
- 记录执行日志
- 错误处理和重试

## 7. 评估规则优化

### 7.1 优化机制

1. **人工反馈收集**：每次人工调整评分时记录原因
2. **规则迭代**：根据反馈调整评分权重和阈值
3. **效果验证**：追踪采纳选题的实际表现，验证评估准确性

### 7.2 评估准确性指标

| 指标 | 定义 | 目标值 |
|------|------|-------|
| 评估通过率 | 评估通过（≥70分）的选题比例 | 20-30% |
| 采纳成功率 | 人工采纳的选题比例 | 50%以上 |
| 预测准确性 | 预测表现与实际表现的匹配度 | >70% |
